其實在七月的時候就正式向原東家(某媒體集團)提出離職。找工作找了快兩個月，當然拉，其實兩個禮拜就找到工作了，但中間遇到亂流。

# 這次找工作的經驗
其實也不是第一次找工作了，只是這次的經驗比較不一樣，同時有兩個offer在考慮，只能說我下次真的不要有這種**選擇困難症**，但是呢，最後居然都不是去這兩間公司之一，原因是透過獵人頭讓我找到一家薪水更優渥的公司......

但悲劇就在這裡發生了。原來這家公司給的薪水優渥不是沒有原因，雖然我自己之前上網也有對這間公司做一點功課、知道他們裡面好像沒有到很好......，但我就是被數字沖昏頭了......

總之，這間公司的團隊氛圍實在是非常的不好。也謝謝這間公司給我這個機會。
> 讓我知道「**團隊氛圍**」對我在找工作的比重其實蠻重要的

最後在兩個禮拜內，八月底前火速提出離職。
這是我人生中第二次沒有做滿一個月的公司。

原本以為經濟不景氣，可能要在待業一陣子...但後來一～二個禮拜還是找到另外一家新東家，還好這裡還可以，讓我待到了現在。

這次的經驗告訴我：
- 不要太相信獵人頭給的資訊，要相信自己做的功課
- 勇敢做選擇
- 沒有對不對、只有適合不適合

而之所以選擇這裡的原因是：
- 可以正式使用 python 作為我工作的主力語言(因為過去都是滿滿的PHP,但同時這對我也是一種**挑戰**)
- 雖然職缺敘述裡面寫道「維護web api」,但實際我在做的是nas裡面的軟體(有點和硬體做擦邊球？)
- 主管算是還不錯，感覺他所建立的團隊氛圍應該不會太差。同時也是在見證新團隊建立的過程，從這過程中吸取他人經驗值
- ~~薪水有達到我設定的數字~~
- ~~距離我目前居住的地方很近~~

當然拉，我自己也覺得我可能也找得太快找得太急。後續有很多面試我後來也都自行排掉了，蠻可惜的。之後希望存夠錢，好好的來面試。

同時，也感謝最近讀的一本書：[全端工程師的職涯筆記](https://www.books.com.tw/products/0010928379)，讓我在面試中準備了簡報，讓我後續的錄取率真的是大增。

# 這兩個月正在學習的新玩意ㄦ
- 對 flask 有更深的認識：學習`flask-restful` ,`blueprint`等等技巧
- 對 docker container 有更深認識： 知道`docker import/export`等等
- 對 linux & shell script有更多認識：因為裡面其實都是寫一堆shell script來啟動container,建立 nas 軟體環境
- 對 qnap nas 有所認識, 知道`qdk`的開發規範：知道nas軟體如何建立，加上前項的東西讓我兜出整個的拼圖
- 對 message queue 有更深認識：由於nas軟體裡面各container是透過MQ溝通，所以多少碰了一些...

另外啊，我最近接受新的任務：要為這個軟體建造與MQTT的橋樑，算是第一次有這個機會與`MQTT`做交手了，好玩

# 換Hugo主題
原本我是使用[hugo-theme-bootstrap](https://github.com/razonyang/hugo-theme-bootstrap)作為本站的佈景主題，但是後來我覺得這個主題實在是太太太複雜了，後來決定用比較簡單的風格，使用[even](https://github.com/olOwOlo/hugo-theme-even)這個主題。目前整體看起來我覺得蠻不錯的＾＾

而且也因為輕量簡潔的關係，我發現我丟到行動版的分數比原本主題還高。至於電腦版...是已經到頂的100分了XD

那換主題的過程中有發現很多的雷，原因主要是**舊的佈景主題寫了很多特別的shortcode,但新的佈景主題不吃**，所以這部分要處理一下。。。

# 前言
去年，ChatGPT開始火紅。火紅到了現在、各式各樣的應用與生活想法都應運而生。同時，很多程式語言也藉由反向工程的方式串接ChatGPT開源讓許多人使用？甚至最近openAI公司也公開api讓市場串接。

不免，我也想要寫一下相關的文章來蹭蹭一下流量（誤

分享一些個人想法看看吧、以及我身為軟體工程師的生存焦慮。

# 工程師要失業了嗎？
其實使用到現在，不免發現他真的可以來寫程式！只要你提出的需求、問題是對的，就會有相對應的 Solution, Code 生成給你，`Prompt engineer`、`Prompt 工程`更因運而生，我自己也常用來比較一些框架的學習、比較，感覺真的非常受用。當然他還不夠完全、畢竟，**人的世界是很複雜的**，有時候還是需要有些已有經驗的專業人士做判斷已確認答案是否完全無誤！這或許，真的會讓我們工程師失業吧？！

不免開始有一些生存焦慮感的說...

然而，當電子書出現紙本書就完全被消滅了嗎？當網際網路出現那些傳統產業的人也完全消滅了嗎？

我認為：新的科技勢必會帶來舊的存在威脅、但同時**也一定會帶來新的問題、新的機會。**

舉例來說：
未來可能資料科學的相關工作會越來越多、如何做出精確的資料清理、資料分析、能餵進對的資料給模型訓練，這或許是一個很重要的AI根基......

# 你敢讓AI完全掌控嗎
不幸的，我們不太可能真的讓AI full control... 為什麼？試想如果AI幫你做司法上面的決策判決，你覺得冤獄冤案的機會就會真的有效減少嗎？AI 裡面有些數學理論去生成、預判，但事實上，**人類是很難被預判的。**，在黑箱的演算法下，你真的敢讓AI完全控制嗎？

有興趣的話我誠摯推薦一本書：[打開演算法黑箱：反噬的AI、走鐘的運算，當演算法出了錯，人類還能控制它嗎？](books.com.tw/products/0010819360)

裡面就有講到類似的事情...或許、我們應該開源開放AI裡面的演算邏輯與規則、以及適當的救濟機制，AI要全面掌控，我認為還有一段距離要走。

AI能提供很好的決策建議，**但執行上仍須人類去過濾、行動。**

# AI 是工具，如何使用它幫助我們的生活更加進步

在這個部分，我覺得最近老高的影片說得很不錯：([傳送門](https://www.youtube.com/watch?v=7uljoeHgN4g))

我覺得ChatGPT出現對於工程師而言，真正厲害的並不再是你會了多少語言、能信手捻來寫出多少厲害的app...。從現在開始，『軟實力』可能才是決定一個工程師的價值吧！例如與其他部門溝通、團隊合作、implement requirement、how to write good code...等等。

願我們能保持開放、樂觀的心態看待新的事物，對未來仍然不要失去繼續嘗試的勇氣與積極。

> **雖然我也是很焦慮、但看一步算一步**

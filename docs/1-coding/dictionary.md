## Dependency Injection
控制權反轉，注入介面使用介面的方法，以便日後維護與替換

> DI 其實是個手段，不是目標。

```
Inversion of Control （控制反轉）是實現低耦合的最佳設計方式之一，讓通用的程式碼來控制應用特定的程式碼，相依於抽象而不倚賴實作。

實現 IoC 的做法有：DI、工廠模式……等。
```

```
DIP 是個使用抽象時依賴關係的準則或概念，IoC 說明了依賴關係的控制方向，而 DI 是一種處理依賴關係的模式。
```

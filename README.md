### Code Review

> Code Review 同樣可以對應心理學中的馬斯洛需求層次理論，協助檢視團隊在不同層面的品質表現。

- 來源：https://www.dein.fr/posts/2015-02-18-maslows-pyramid-of-code-review
- Correct 正確性 ➝ 基本生理需求
- Secure 安全 ➝ 安全需求
- Readable 可讀性 ➝ 愛與歸屬需求
- Elegant 優雅 ➝ 尊嚴需求
- Altruist 能否幫助他人（利他主義）➝ 自我實現

### 認知負荷才是最重要的

- 來源：https://github.com/zakirullin/cognitive-load/blob/main/README.zh-cn.md

```text
這世上有如此多的「流行語」和「最佳實踐」，但我們更應該關注根本：開發人員在瀏覽程式碼時所感受到的困惑程度。

困惑會帶來時間與金錢的代價。困惑源於高認知負荷，這不是抽象概念，而是人類的基本限制。

由於閱讀與理解程式碼的時間遠超過撰寫程式碼，我們必須持續檢視自己是否在程式中放入過多會增加認知負荷的內容。「認知負荷」（Cognitive load）指的是開發者為完成任務所需思考的總量。

閱讀程式碼時，人們會在腦中暫存變數數值、控制流程、呼叫順序等資訊。一般人在工作記憶中約能維持四個片段。一旦超過上限，理解就會變得困難。

想像我們被要求修復一個完全陌生的專案，並得知前任開發者極為聰明，採用了大量高階架構、炫麗函式庫與先進技術。換句話說，前任開發者替我們留下了極高的認知負荷。
```
